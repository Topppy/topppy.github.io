js 判断数组是否包含目标值，使用了 in 操作符 （窝自己的理解误区）

2018.03.27

一般，我们会使用indexOf,就像下面这样

console.log( [1,2,6,7].indexOf(6) > -1)    
// true

据说还可以使用in操作符，于是试了一下

console.log(  1 in [1,2,6,7])    
// true
console.log(  2 in [1,2,6,7])    
// true

嗯感觉没问题。。。然后就悲剧了

console.log(  6 in [1,2,6,7])    
// false
console.log(  7 in [1,2,6,7])    
// false

赶紧查文档 mdn

in 操作符实际上检测的是 prop in object，也就是说，在对数组使用in的时候，检测的是key是index。。。。而不是对应的value。所以对于长度为4的数组[1,2,6,7]，0、1、2、3 都返回 true， 大于3的都是false。

那么还有别的方法吗？ES6 的arr.includes()

[1,2,6,7].includes(6)
// true
